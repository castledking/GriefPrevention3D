v18.0.1: Legacy Compatibility Fixes & Release Automation
# GriefPrevention3D v18.0.1

**Wiki:** [GriefPrevention3D Wiki](https://github.com/castledking/GriefPrevention3D/wiki?utm_source=chatgpt.com)

## Highlights

- **Fixed additional 1.8.8 runtime crashes** — Removed more direct references to modern Bukkit enum values from active legacy event paths.
- **Added `/gpdebug`** — Server admins can now toggle debug logging without editing config.yml or running `/gpreload`.
- **Added release safeguards** — Gradle now fails verification if `gradle.properties` and `pom.xml` versions drift apart.
- **Prepared CurseForge automation** — Release workflows now include an optional CurseForge upload target for the GP3D project.

## Fixes

### Legacy Servers (1.8.8)

- Fixed `EntityDamageByEntityEvent` and `EntityCombustByEntityEvent` errors caused by direct `AREA_EFFECT_CLOUD` enum access.
- Fixed related modern entity checks for shulkers, tridents, raiders, vexes, and evoker fangs so they do not crash older Bukkit runtimes.
- Fixed `EntityInteractEvent` crop-trample handling on 1.8.8 by supporting the legacy `SOIL` material name alongside modern `FARMLAND`.
- Fixed restore-nature player block classification for farmland across legacy and modern material names.

## Admin Tools

- Added `/gpdebug`.
- Added `griefprevention.debug`, defaulting to operators and included under `griefprevention.admin.*`.
- `/gpdebug` toggles `GriefPrevention.Abridged Logs.Included Entry Types.Debug` immediately and persists it to config.yml.
- `/gpdebug on` and `/gpdebug off` are supported for explicit state changes.

## Release Workflow

- Added a Gradle `checkPomVersion` verification task to protect addon compatibility metadata in `pom.xml`.
- Wired `checkPomVersion` into Gradle's normal `check` lifecycle.
- Added CurseForge release automation for project `1552252`.
- Added workflow input `run_curseforge` so CurseForge publishing can be enabled or disabled during manual release runs.
