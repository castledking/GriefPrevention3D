v18.1.6: Minimum distance claim creation, neighbor trust, granular permissions
# GriefPrevention3D v18.1.6

**Full Release Notes:** [v18.1.6.md](https://github.com/castledking/GriefPrevention3D/blob/master/.release/latest/v18.1.6-full.md)

## Highlights

- **Minimum Distance Between Claims** — New config option `MinimumDistance` (default 0 = disabled). When set, players can't create claims within the configured distance of other claims. Conflicting claims are visualized with a CONFLICT_ZONE boundary.
- **Neighbor Trust** — New trust level that lets specific players bypass minimum distance. Use `/claim trust <player> neighbor` or `/neighbortrust <player>`. Toggle per-claim allow-all with `/claim distance toggle`.
- **Auto-Neighbor Trust** — Existing claims too close together automatically grant neighbor trust to each other. Auto-removed when claims are abandoned.
- **Granular Permissions** — 27 individual permissions (e.g. `griefprevention.neighbortrust`, `griefprevention.checkclaimdistance`) now available under `griefprevention.claims`. All default to `true` via parent permission.
- **New Commands** — `/claim distance check` (visualize nearby claims), `/claim distance toggle` (per-claim bypass), `/neighbortrust` (alias: `/distancetrust`).
- **Translations** — Spanish (`messages_es.yml`) and Portuguese-BR (`messages_pt_BR.yml`) updated with all new messages.
