v17.4.8: Fix WelcomeTask memory leak and allow disabling
# GriefPrevention3D v17.4.8

**Wiki:** https://github.com/castledking/GriefPrevention3D/wiki

## Highlights

- **Fixed WelcomeTask memory leak** ([#2483](https://github.com/GriefPrevention/GriefPrevention/issues/2483)) — The scheduled welcome task no longer holds a hard reference to the `Player` object, preventing leaked `CraftPlayer` instances when players disconnect before the task runs.
- **Disable welcome task via config** — Setting `ManualDeliveryDelaySeconds` to `0` or a negative value now skips scheduling the welcome task entirely.

## WelcomeTask Memory Leak Fix

`WelcomeTask` previously stored a direct `Player` reference. On high-volume servers with a large `ManualDeliveryDelaySeconds` value, every new player who joined and left before the task fired would leave a leaked `CraftPlayer` in memory until the task eventually ran.

The task now stores the player's `UUID` and resolves it via `Bukkit.getPlayer()` at execution time. If the player is offline, the task exits immediately with no retained references.

## Disabling the Welcome Task

Servers that don't want the welcome message can now set `ManualDeliveryDelaySeconds` to `0` (or any negative value) in `config.yml` to prevent the task from being scheduled at all. Previously, some server operators worked around this by setting extremely high delay values, which caused the leak described above.

## Migration

No data migration required. Existing `ManualDeliveryDelaySeconds` values continue to work as before. Set to `0` to disable.
