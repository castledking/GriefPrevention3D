v17.4.9: Fix thin 3D claim resize regression
# GriefPrevention3D v17.4.9

**Wiki:** https://github.com/castledking/GriefPrevention3D/wiki

## Highlights

- **Fixed a resize selection regression** - Very small 3D subdivisions and 3D admin claims can be right-clicked with the golden shovel again to start resize selection.
- **Restored thin claim corner clicks** - Claims with a one-block X, Y, or Z dimension no longer silently consume the shovel click without showing the selection message.
- **Preserved shaped-claim behavior** - The fix restores rectangular claim corner handling without changing shaped-claim corner selection.

## Regression Fix

Recent corner-selection changes regressed resize selection for thin 3D claims. Affected claims could consume the golden shovel right-click without sending the resize/selection message, making them appear impossible to resize.

This affected 3D subdivisions and 3D admin claims with any one-block dimension, including `1x1x1`, `1x1x2`, `1x2x1`, and single-layer `2x2x1` claims. The selection path now recognizes rectangular corners directly, even when the rectangle collapses to a single X or Z coordinate, and shovel targeting resolves visible single-layer markers back to the real claim corner.

## Technical Changes

- Added regression coverage for one-block X/Z rectangular corner detection.
- Kept shaped-claim corner detection on the existing polygon path.
- This is not gated by `GriefPrevention.Claims.AllowShapedClaims`; the affected corner-selection path is shared by normal rectangular claim resizing.

## Migration

No data migration or configuration changes required.
